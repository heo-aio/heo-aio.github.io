# 🛠️ Hugo + GitHub Pages 배포 삽질기: "테마가 안 바뀌어요!" 해결 일지

최근 Hugo로 블로그를 이전하면서 테마를 Stack에서 PaperMod로 변경하는 과정에서 겪은 고난과 역경을 기록한다.

결론부터 말하면 **범인은 파일 이름과 서브모듈 설정이었다.** 😅

---

## 🔍 문제 발생: 테마 변경 후에도 그대로인 화면

분명 `hugo.toml`을 수정하고 PaperMod로 테마를 변경했는데, 브라우저에는 여전히 예전 테마인 Stack이 나타나는 현상이 발생했다.

### 1차 시도

GitHub Pages를 Unpublish 해보았다.

> 📌 결과
> 사이트만 내려갈 뿐, 다시 배포해도 동일한 화면이 나타났다.
>
> 즉, 시스템 문제가 아니라 **빌드 내용 자체의 문제**였다.

---

## 🕵️ 범인 찾기 1: 파일 이름 (확장자)

Hugo는 설정 파일의 이름을 매우 엄격하게 관리한다.

### ⚠️ 삽질 포인트

루트 디렉토리에 있는 설정 파일 이름이 단순히 `hugo`로 되어 있었다.

### 🚀 해결 방법

파일명을 아래처럼 수정했다.

```text
hugo → hugo.toml
```

### 💡 교훈

컴퓨터에게 파일의 용도를 알려주는 확장자(`.toml`, `.yaml`)는 매우 중요하다.

> ⚠️ 주의
>
> 윈도우에서 파일 확장자가 숨겨진 상태라면 이름을 변경할 때
>
> ```text
> hugo.toml.toml
> ```
>
> 과 같이 중복 확장자가 생길 수 있다.
>
> 탐색기의 **[파일 확장명] 옵션을 활성화**하고 작업하는 것이 안전하다.

---

## 🕵️ 범인 찾기 2: GitHub Actions 워크플로우 파일

GitHub Actions가 빌드를 제대로 수행하지 못하고 있었다.

### ⚠️ 삽질 포인트

`.github/workflows/` 폴더 안의 파일 이름도 단순히 `hugo`로 되어 있었다.

### 🚀 해결 방법

파일명을 아래처럼 수정했다.

```text
hugo → hugo.yml
```

이후 GitHub Actions가 정상적으로 워크플로우를 인식할 수 있게 되었다.

---

## 🕵️ 범인 찾기 3: 서브모듈(Submodule)의 늪

이번 문제의 핵심 원인이었다.

### 📌 원인 분석

테마 파일들은 로컬 PC에는 존재했지만 GitHub 서버에는 껍데기 정보만 올라가 있었다.

그 결과 빌드 과정에서 테마를 찾지 못했고, 빌드 로그에도 다음과 같은 문구가 출력되지 않았다.

```text
Themes: PaperMod
```

### ⚠️ 삽질 포인트

서브모듈이 제대로 연결되지 않은 상태였다.

### 🚀 해결 방법

주소를 동기화하고 실제 파일을 내려받았다.

```bash
git submodule sync
```

```bash
git submodule update --init --recursive
```

이후 테마 폴더를 추가했다.

```bash
git add themes/PaperMod
```

정상적으로 반영된 후에는 GitHub에서도 PaperMod 테마를 인식하기 시작했다.

## ✅ 최종 해결 체크리스트

혹시 나중에 같은 문제가 발생한다면 아래 순서대로 확인하자.

### 1. 파일 확장자 확인

```text
hugo.toml
hugo.yml
```

이 정확하게 존재하는가?

---

### 2. 빌드 로그 확인

GitHub 저장소의 **Actions 탭**에서 아래 문구가 출력되는가?

```text
Themes: PaperMod
```

---

### 3. 브라우저 캐시 확인

시크릿 모드(`Ctrl + Shift + N`)에서도 동일한 현상이 발생하는가?

> 📌 참고
>
> 시크릿 모드에서도 동일하다면 캐시 문제가 아니라 서버 또는 빌드 문제일 가능성이 높다.

---

### 4. public 폴더 정리

배포 시 충돌을 방지하기 위해 로컬의 `public/` 폴더는 삭제하고 `.gitignore`에 등록하는 것이 좋다.

```gitignore
public/
```

## 📝 후기

이번 삽질을 통해 한 가지 사실을 다시 깨달았다.

> 💡 배운 점
>
> 컴퓨터는 내가 말한 대로 움직이는 것이 아니라,
> **내가 설정한 대로 움직인다.**

작은 파일 이름 하나, 확장자 하나, 서브모듈 설정 하나가 전체 배포 결과를 바꿔버릴 수 있다는 것을 몸소 경험했다.

이제는 블로그가 정상적으로 배포되니, 테마 때문에 고민하던 시간만큼 더 열심히 글을 작성해 볼 생각이다. ✍️

이 기록이 Hugo로 블로그를 시작하는 다른 분들에게 도움이 되길 바란다. 😊
