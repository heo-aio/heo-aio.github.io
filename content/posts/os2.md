# 01. 📖 정적 할당과 동적 할당

## 개념

프로그램이 메모리를 사용하는 방식은 크게 정적 할당(Static Allocation)과 동적 할당(Dynamic Allocation)으로 구분된다.

둘의 가장 큰 차이는 메모리를 확보하는 시점이다.

---

## 원리

### 정적 할당 (Static Allocation)

프로그램 실행 전에 필요한 메모리를 미리 확보하는 방식이다.

주로 Data 영역에 저장되는 전역 변수와 정적 변수에서 사용된다.

#### 장점

✅ 빠른 접근 속도

✅ 관리가 단순함

#### 단점

❌ 메모리 낭비 가능

❌ 실행 중 크기 변경 불가

---

### 동적 할당 (Dynamic Allocation)

프로그램 실행 중 필요한 시점에 메모리를 할당하는 방식이다.

주로 Heap 영역에서 이루어진다.

#### 장점

✅ 필요한 만큼만 사용 가능

✅ 유연한 메모리 관리

#### 단점

❌ 할당 및 해제 비용 발생

❌ 메모리 누수 가능

---

## 예시

```c
int global_var = 10; // 정적 할당

int* ptr = malloc(sizeof(int)); // 동적 할당
```

---

## 💼 면접 포인트

**Q. Heap 영역과 Stack 영역의 차이는?**

> Stack은 함수 호출 시 자동으로 생성·해제되고, Heap은 실행 중 동적으로 할당 및 해제된다.

# 02. 🔄 프로세스의 5가지 상태

## 개념

프로세스는 생성부터 종료까지 여러 상태를 거치며 실행된다.

---

## 원리

```text
New
 ↓
Ready
 ↓
Running
 ↙   ↘
Waiting  Ready
 ↓
Ready
 ↓
Running
 ↓
Terminated
```

---

### New

프로세스 생성 단계

---

### Ready

CPU를 기다리는 상태

---

### Running

CPU를 할당받아 실행 중인 상태

---

### Waiting (Blocked)

I/O 작업 완료를 기다리는 상태

---

### Terminated

실행이 종료된 상태

---

## 예시

웹 브라우저에서 파일 다운로드를 수행한다고 가정해보자.

1. 브라우저 실행 → Ready
2. CPU 할당 → Running
3. 파일 다운로드 요청 → Waiting
4. 다운로드 완료 → Ready
5. CPU 재할당 → Running

---

## 💼 면접 포인트

**Q. Running 상태와 Waiting 상태의 차이는?**

> Running은 CPU를 사용 중인 상태이고, Waiting은 특정 이벤트(I/O 완료 등)를 기다리는 상태이다.

---

# 03. 🔗 멀티 프로세스와 IPC

## 개념

운영체제는 여러 프로세스를 동시에 실행할 수 있으며, 필요에 따라 프로세스 간 데이터를 주고받아야 한다.

이를 위해 IPC(Inter Process Communication)를 제공한다.

---

## 원리

### 멀티 프로세스

여러 개의 독립적인 프로세스를 실행하는 방식

#### 장점

✅ 높은 안정성

✅ 프로세스 간 영향 최소화

#### 단점

❌ 생성 비용 큼

❌ 통신 복잡

---

### IPC

독립적인 프로세스 간 데이터 교환 기술

대표적인 방식은 다음과 같다.

| 방식            | 설명        |
| ------------- | --------- |
| Message Queue | 메시지 기반 통신 |
| Shared Memory | 메모리 공유    |
| Socket        | 네트워크 통신   |

---

## 예시

웹 서버와 데이터베이스 서버는 별도의 프로세스로 동작한다.

이때 Socket을 통해 요청과 응답을 주고받는다.

---

## 💼 면접 포인트

**Q. 프로세스끼리는 왜 IPC가 필요한가?**

> 프로세스는 기본적으로 독립된 메모리 공간을 사용하므로 직접 데이터를 공유할 수 없기 때문이다.

---

# 04. ⚙️ CPU 스케줄링

## 개념

CPU는 한정된 자원이기 때문에 어떤 프로세스를 먼저 실행할지 결정해야 한다.

이 과정을 CPU 스케줄링이라고 한다.

---

## 원리

### FCFS

First Come First Served

먼저 도착한 작업을 먼저 처리한다.

#### 장점

✅ 구현 단순

#### 단점

❌ Convoy Effect 발생 가능

---

### SJF

Shortest Job First

실행 시간이 가장 짧은 작업 우선 처리

#### 장점

✅ 평균 대기시간 최소화

#### 단점

❌ 긴 작업이 계속 밀릴 수 있음

---

### Round Robin

일정 시간(Time Quantum)마다 CPU를 교체한다.

#### 장점

✅ 공정성 우수

✅ 응답성 우수

#### 단점

❌ Context Switching 증가

---

## 예시

프로세스 A, B, C가 존재할 때

```text
A → B → C → A → B → C
```

Round Robin 방식에서는 CPU를 순환하며 사용한다.

---

## 💼 면접 포인트

**Q. Round Robin이 현대 운영체제에서 많이 사용되는 이유는?**

> 특정 프로세스가 CPU를 독점하지 못하며 사용자 응답성이 우수하기 때문이다.

---

# 05. 🚨 Deadlock과 Starvation

## 개념

멀티 프로세스 환경에서는 자원 경쟁으로 인해 다양한 문제가 발생할 수 있다.

대표적인 문제가 Deadlock과 Starvation이다.

---

## 원리

### Deadlock

프로세스들이 서로의 자원을 기다리며 무한 대기에 빠진 상태

#### 발생 조건

* 상호배제
* 점유대기
* 비선점
* 순환대기

---

### Starvation

특정 프로세스가 계속 자원을 할당받지 못하는 상태

---

## 예시

### Deadlock

```text
프로세스 A → 자원 B 대기

프로세스 B → 자원 A 대기
```

서로 기다리며 무한 정지

---

### Starvation

높은 우선순위 프로세스가 계속 실행되어

낮은 우선순위 프로세스가 CPU를 받지 못하는 상황

---

## 💼 면접 포인트

**Q. Deadlock과 Starvation의 차이는?**

> Deadlock은 서로가 서로를 기다리는 상태이고, Starvation은 특정 프로세스만 지속적으로 자원을 받지 못하는 상태이다.

---

# 📝 핵심 정리

✅ 프로세스는 정적 할당과 동적 할당을 통해 메모리를 관리한다.

✅ 프로세스는 New → Ready → Running → Waiting → Terminated 상태를 반복한다.

✅ 독립적인 프로세스 간 통신을 위해 IPC가 사용된다.

✅ CPU 스케줄링은 한정된 CPU 자원을 효율적으로 분배하기 위한 기법이다.

✅ Deadlock은 상호 대기 상태, Starvation은 특정 프로세스의 지속적인 자원 부족 상태이다.

---

> 운영체제의 핵심 역할은 CPU와 메모리 같은 제한된 자원을 효율적으로 관리하는 것이다. 프로세스 상태 전이, IPC, 스케줄링은 모두 이러한 목표를 달성하기 위한 핵심 메커니즘이라고 볼 수 있다.
