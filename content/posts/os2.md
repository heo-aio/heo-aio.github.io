# 🔄 운영체제 핵심 개념 정리 - 프로세스 상태 전이, IPC, 스케줄링

지난 글에서는 프로세스(Process)의 개념과 메모리 구조를 정리했다.

이번에는 프로세스가 실제로 실행되면서 어떤 상태를 거치는지, 프로세스끼리는 어떻게 통신하는지, CPU는 어떤 기준으로 작업을 처리하는지에 대해 정리해보려고 한다.

운영체제를 공부하다 보면 용어가 정말 많이 나오는데, 결국 핵심은 하나인 것 같다.

> 운영체제는 제한된 CPU와 메모리 자원을 효율적으로 관리하기 위해 존재한다.

이번 글에서 나오는 프로세스 상태 전이, IPC, CPU 스케줄링, Deadlock 역시 모두 이 목적을 위해 만들어진 개념들이다.

---

## 📌 목차

* 정적 할당과 동적 할당
* 프로세스의 5가지 상태
* 멀티 프로세스와 IPC
* CPU 스케줄링
* Deadlock과 Starvation
* 공부하면서 느낀 점

# 01. 📖 정적 할당과 동적 할당

프로그램이 메모리를 사용하는 방식은 크게 정적 할당(Static Allocation)과 동적 할당(Dynamic Allocation)으로 나뉜다.

처음에는 둘 다 그냥 메모리를 사용하는 방법이라고 생각했는데, 가장 큰 차이는 **언제 메모리를 확보하느냐** 에 있었다.

---

## 🔹 정적 할당 (Static Allocation)

프로그램이 실행되기 전에 필요한 메모리를 미리 확보하는 방식이다.

주로 Data 영역에 저장되는 전역 변수나 정적 변수에서 사용된다.

### 장점

✅ 빠른 접근 속도

✅ 관리가 단순함

### 단점

❌ 메모리 낭비 가능

❌ 실행 중 크기 변경 불가

---

## 🔹 동적 할당 (Dynamic Allocation)

프로그램 실행 중 필요한 시점에 메모리를 할당하는 방식이다.

주로 Heap 영역에서 이루어진다.

### 장점

✅ 필요한 만큼만 사용 가능

✅ 유연한 메모리 관리

### 단점

❌ 할당 및 해제 비용 발생

❌ 메모리 누수 가능

---

## 예시

```c
int global_var = 10; // 정적 할당

int* ptr = malloc(sizeof(int)); // 동적 할당
```

---

## 🤔 공부하면서 정리한 내용

운영체제를 처음 공부할 때 Heap과 Stack이 가장 헷갈렸다.

둘 다 데이터를 저장하는 공간이라고만 생각했는데,

다시 정리해보니 핵심은

* Stack → 자동 관리
* Heap → 직접 관리

였다.

그래서 동적 할당을 사용할 때 메모리 해제를 하지 않으면 메모리 누수가 발생하는 것이다.

---

# 02. 🔄 프로세스의 5가지 상태

프로세스는 생성부터 종료까지 여러 상태를 거치며 실행된다.

책에서는 상태 다이어그램으로 설명하는 경우가 많은데, 직접 따라가 보니 훨씬 이해하기 쉬웠다.

---

## 상태 전이 과정

```text
New
 ↓
Ready
 ↓
Running
 ↙   ↘
Waiting  Ready
 ↓
Ready
 ↓
Running
 ↓
Terminated
```

---

### New

프로세스가 생성된 상태

---

### Ready

CPU를 할당받기 위해 기다리는 상태

---

### Running

CPU를 할당받아 실제 명령어를 수행하는 상태

---

### Waiting (Blocked)

I/O 작업이나 특정 이벤트를 기다리는 상태

---

### Terminated

프로세스 실행이 종료된 상태

---

## 예시

웹 브라우저에서 파일 다운로드를 수행한다고 가정해보자.

```text
브라우저 실행
↓
Ready

CPU 할당
↓
Running

파일 다운로드 요청
↓
Waiting

다운로드 완료
↓
Ready

CPU 재할당
↓
Running
```

---

## 🤔 처음에 헷갈렸던 부분

Ready 상태와 Waiting 상태가 처음에는 비슷하게 느껴졌다.

둘 다 실행되지 않는 상태처럼 보였기 때문이다.

하지만 다시 보니 차이가 있었다.

| 상태      | 의미          |
| ------- | ----------- |
| Ready   | CPU만 기다림    |
| Waiting | 특정 이벤트를 기다림 |

즉,

Ready는 실행 준비가 끝난 상태이고,

Waiting은 아직 기다려야 할 작업이 남아 있는 상태이다.

---

# 03. 🔗 멀티 프로세스와 IPC

컴퓨터를 사용하다 보면 브라우저, 메신저, 음악 플레이어를 동시에 실행하는 일이 많다.

이처럼 여러 프로세스를 동시에 실행하는 환경을 멀티 프로세스라고 한다.

---

## 🔹 멀티 프로세스

여러 개의 독립적인 프로세스를 실행하는 방식

### 장점

✅ 높은 안정성

✅ 프로세스 간 영향 최소화

### 단점

❌ 생성 비용 큼

❌ 통신 복잡

---

## 🔹 IPC (Inter Process Communication)

프로세스는 기본적으로 독립된 메모리 공간을 사용한다.

그래서 서로 데이터를 주고받기 위해서는 별도의 통신 방식이 필요하다.

이를 IPC라고 한다.

---

### 대표적인 IPC 방식

| 방식            | 설명        |
| ------------- | --------- |
| Message Queue | 메시지 기반 통신 |
| Shared Memory | 메모리 공유    |
| Socket        | 네트워크 통신   |

---

## 예시

웹 서버와 데이터베이스 서버는 일반적으로 별도의 프로세스로 실행된다.

이때 Socket을 이용하여 데이터를 주고받는다.

---

## 🤔 내가 이해한 방식

처음에는

```text
프로세스가 여러 개 있으면 알아서 데이터 공유하는 거 아닌가?
```

라고 생각했다.

하지만 프로세스는 서로 독립적인 메모리 공간을 사용한다.

그래서 데이터를 주고받으려면 IPC 같은 별도의 통신 방식이 필요한 것이다.

---

# 04. ⚙️ CPU 스케줄링

CPU는 한 번에 하나의 작업만 처리할 수 있다.

그런데 실제 컴퓨터에서는 여러 프로그램이 동시에 실행되는 것처럼 보인다.

이때 운영체제가

> 누구에게 CPU를 먼저 줄 것인가?

를 결정하는 과정을 CPU 스케줄링이라고 한다.

---

## 🔹 FCFS

First Come First Served

먼저 도착한 작업을 먼저 처리한다.

### 장점

✅ 구현 단순

### 단점

❌ Convoy Effect 발생 가능

---

## 🔹 SJF

Shortest Job First

실행 시간이 가장 짧은 작업을 먼저 처리한다.

### 장점

✅ 평균 대기시간 최소화

### 단점

❌ 긴 작업이 계속 밀릴 수 있음

---

## 🔹 Round Robin

일정 시간(Time Quantum)마다 CPU를 교체한다.

### 장점

✅ 공정성 우수

✅ 응답성 우수

### 단점

❌ Context Switching 증가

---

## 예시

```text
A → B → C → A → B → C
```

Round Robin 방식에서는 CPU를 순환하면서 사용한다.

---

## 🤔 공부하면서 든 생각

왜 현대 운영체제에서 Round Robin 계열 방식이 많이 사용되는지 조금 이해가 갔다.

만약 하나의 프로세스가 CPU를 계속 사용한다면 다른 프로그램들은 응답하지 못할 것이다.

그래서 CPU를 조금씩 나눠 사용하는 방식이 사용자 입장에서도 훨씬 자연스럽다.

---

# 05. 🚨 Deadlock과 Starvation

운영체제를 공부하면 반드시 등장하는 개념이다.

처음에는 둘 다 비슷해 보였는데 실제로는 원인이 다르다.

---

## 🔹 Deadlock

프로세스들이 서로의 자원을 기다리며 무한 대기에 빠진 상태

### 발생 조건

* 상호배제
* 점유대기
* 비선점
* 순환대기

---

### 예시

```text
프로세스 A → 자원 B 대기

프로세스 B → 자원 A 대기
```

서로 기다리기만 하면서 아무 작업도 진행되지 않는다.

---

## 🔹 Starvation

특정 프로세스가 계속 자원을 할당받지 못하는 상태

---

### 예시

```text
높은 우선순위 프로세스 계속 실행

↓

낮은 우선순위 프로세스 실행 기회 없음
```

---

## 🤔 처음에는 둘이 같은 개념인 줄 알았다

둘 다 실행이 안 되는 상황이라고 생각했기 때문이다.

하지만 다시 보니 차이가 있었다.

| 구분         | 설명                |
| ---------- | ----------------- |
| Deadlock   | 서로 기다리면서 모두 멈춤    |
| Starvation | 특정 프로세스만 계속 실행 못함 |

그래서 나는

```text
Deadlock = 전체가 막힘

Starvation = 특정 작업만 굶음
```

정도로 기억하고 있다.

---

# 📝 공부하면서 느낀 점

운영체제를 공부하면서 느끼는 건 결국 모든 개념이 연결되어 있다는 점이다.

프로세스 상태 전이도 CPU를 효율적으로 사용하기 위한 것이고,

스케줄링도 CPU를 공정하게 분배하기 위한 것이며,

IPC 역시 여러 프로세스가 협력하기 위해 필요한 기술이다.

처음에는 각각의 개념이 따로 노는 것처럼 보였는데,

계속 공부하다 보니 결국

> "제한된 자원을 효율적으로 관리하기 위한 방법"

이라는 공통된 목적을 가지고 있다는 것을 알게 되었다.

아직 운영체제는 어렵지만 프로세스 → 스레드 → 동기화 → 교착상태 순으로 연결해서 공부하면 조금씩 흐름이 보이는 것 같다.

---

# 🚀 주기적으로 다시 보기

* 정적 할당 = 실행 전 메모리 확보
* 동적 할당 = 실행 중 메모리 확보
* Ready = CPU 대기
* Waiting = 이벤트 대기
* IPC = 프로세스 간 통신
* FCFS = 먼저 온 작업 먼저 처리
* SJF = 짧은 작업 우선 처리
* Round Robin = 시간 단위로 CPU 순환
* Deadlock = 서로 기다리며 멈춤
* Starvation = 특정 프로세스만 계속 실행 못함

운영체제는 암기할 내용이 많지만, 각각의 개념이 왜 필요한지 이해하면서 공부하면 조금 더 오래 기억에 남는 것 같다.
