# 📌 왜 NumPy부터 잡는가

Pandas의 데이터프레임도, 딥러닝 프레임워크의 텐서도 결국 내부는 NumPy 배열과 같은 방식으로 동작한다.

```text
NumPy 배열 (ndarray)

↓

Pandas DataFrame / Series

↓

머신러닝 · 딥러닝 입력 데이터
```

그래서 배열의 **모양(shape)** 과 **축(axis)**, 그리고 **메모리 동작**을 이해하면 그 위의 모든 라이브러리가 훨씬 쉬워진다.

---

# 1️⃣ 배열 생성과 인덱싱 기초

파이썬 리스트에서 `np.array`로 배열을 만든다. 한 배열의 원소는 보통 같은 타입으로 맞춰진다(동질성).

```python
import numpy as np

a = np.array([1, 2, 3, 4, 5, 6])
print(a.dtype)   # int64
print(a[0])      # 첫 원소 (0-based 인덱싱)
print(a[-1])     # 마지막 원소 (음수 인덱스)
```

여기까지는 파이썬 리스트와 거의 같아서 무리 없이 넘어갔다.

문제는 슬라이스부터였다.

---

# 2️⃣ 뷰(view)와 복사(copy) — 원본이 바뀐다

슬라이스가 원본과 **메모리를 공유**한다는 점을 처음 보고 당황했다.

파이썬 리스트 감각으로는 `b = a[:3]`이 당연히 새로운 배열을 복사해 온 것이라고 생각했다.

```python
a = np.array([1, 2, 3, 4, 5])
b = a[:3]     # 뷰(view) — 원본과 메모리 공유
b[0] = 99
print(a)      # [99  2  3  4  5]  ← 원본이 함께 바뀐다
```

`b`만 수정했는데 `a`가 바뀌어 있어서 한참 코드를 다시 봤다.

값을 진짜로 복사하려면 `.copy()`를 명시적으로 호출해야 한다.

```python
b = a[:3].copy()   # 이제 b를 바꿔도 a는 그대로다
```

`flatten()`은 복사본을 만들고 `ravel()`은 가능한 경우 뷰를 만든다는 것도 같은 맥락이라 함께 외웠다.

> **핵심 정리**
>
> 슬라이스는 기본적으로 뷰(view)이며 원본과 메모리를 공유한다. 전처리 중 원본을 보존하려면 반드시 `.copy()`로 복사해야 한다.

---

# 3️⃣ 축(axis) — 사라지는 축으로 이해하기

집계 함수의 `axis` 인자가 계속 헷갈렸다.

`sum(axis=0)`이 행 방향인지 열 방향인지 매번 헷갈려서, 결국 **"axis는 사라지는 축"** 이라고 이해하기로 했다.

```python
a = np.array([[1, 2],
              [3, 4],
              [5, 6]])   # shape (3, 2)

a.sum(axis=0)   # [ 9 12 ]     ← 0번 축(행)이 사라짐 → 열별 합
a.sum(axis=1)   # [ 3  7 11 ]  ← 1번 축(열)이 사라짐 → 행별 합
```

"어느 축이 없어지는가"로 생각하니 그제서야 헷갈리지 않았다.

`max`, `min`, `mean`, `std`도 모두 같은 규칙을 따른다.

---

# 4️⃣ 브로드캐스팅과 벡터화

모양이 다른 배열끼리 자동으로 맞춰 계산해 주는 것이 브로드캐스팅이다.

예를 들어 `(3, 2)` 배열에 `(1, 2)` 행 하나를 더하면, 그 행이 모든 행에 더해진다.

```python
X = np.array([[1, 2],
              [3, 4],
              [5, 6]])
row = np.array([10, 20])   # shape (2,)
X + row
# [[11 22]
#  [13 24]
#  [15 26]]
```

이 규칙을 알고 나니 반복문을 돌리지 않고 배열을 통째로 계산하는 **벡터화**가 자연스러워졌다.

평균 제곱 오차(MSE)도 반복문 없이 한 줄로 끝난다.

$$
MSE = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)^2
$$

```python
mse = np.mean((y_pred - y_true) ** 2)
```

이 한 줄을 보고 나서, 왜 다들 반복문 대신 벡터 연산을 쓰라고 하는지 몸으로 이해했다.

---

# 5️⃣ 조건으로 값 고르기 — 불리언 인덱싱

값을 "삭제"하는 대신 조건에 맞는 값만 "선택"하는 사고방식도 익혔다.

```python
a = np.array([1, 5, 2, 8, 3])

a[a < 5]                 # [1 2 3]  조건에 맞는 원소만
a[(a > 1) & (a < 8)]     # 복합 조건은 괄호 필수, & 사용
np.where(a < 5, 0, a)    # 조건에 따라 값 치환
np.clip(a, 2, 6)         # 범위를 벗어난 값을 잘라냄
```

Pandas의 조건 필터와 원리가 똑같아서, 여기서 잡아두니 나중에 Pandas가 편했다.

# 6️⃣ 정형 데이터 가공 (실전에 가장 가까웠던 파트)

마지막 파트에서는 서버 점검 CSV를 한 번만 `read_csv`로 읽어 `to_numpy()`로 바꾼 뒤, 그다음부터는 순수 NumPy만으로 가공했다.

전처리에서 실제로 자주 하는 작업들이라 가장 실무 냄새가 났다.

## 열 방향 표준화 (z-score)

$$
z = \frac{x - \mu}{\sigma}
$$

```python
mean = X.mean(axis=0)
std  = X.std(axis=0)
X_std = (X - mean) / std     # 열 단위 표준화
```

## 범주 → 정수 라벨 인코딩

```python
labels, encoded = np.unique(cat, return_inverse=True)
# encoded 에 각 원소의 정수 라벨이 담긴다
```

## train / valid 분할

```python
idx = np.random.permutation(len(X))   # 인덱스를 섞고
cut = int(len(X) * 0.8)
train_idx, valid_idx = idx[:cut], idx[cut:]
```

데이터를 섞지 않고 그냥 앞뒤로 자르면 편향이 생긴다는 이유까지 함께 이해했다.

이 외에 `np.argsort`로 조치 시간 상위를 뽑거나, `np.column_stack`으로 절편·다항항을 붙여 설계행렬을 만드는 것도 다뤘다.

sklearn 없이도 표준화와 라벨 인코딩이 원리대로 되는 것을 직접 본 느낌이라 좋았다.

# 💡 공부하면서 느낀 점

처음에는 NumPy를 그냥 "함수를 외우는 것"이라고 생각했다.

하지만 예제를 직접 풀어 보니 중요한 것은 함수 이름이 아니라, **배열의 모양과 메모리가 어떻게 동작하는지를 이해하는 것**이라는 점을 알게 되었다.

특히 뷰와 복사를 구분하지 않으면 원본이 조용히 오염된다는 것, 그리고 반복문 대신 벡터 연산으로 사고해야 한다는 것이 가장 크게 남았다.

이 기본기가 잡히니 그 위에 Pandas와 시각화를 얹는 것이 훨씬 수월했다.
