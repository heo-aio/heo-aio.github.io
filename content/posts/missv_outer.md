머신러닝 프로젝트에서 모델의 성능을 결정하는 가장 중요한 요소 중 하나는 **데이터 품질(Data Quality)** 이다.

아무리 좋은 알고리즘을 사용하더라도 데이터 자체에 문제가 있다면 좋은 성능을 기대하기 어렵다.

실제 프로젝트에서는 모델을 학습하기 전에 **결측치(Missing Value)** 와 **이상치(Outlier)** 를 처리하는 데이터 전처리 과정을 반드시 거친다.

이번 글에서는 Titanic 데이터셋을 활용하여 결측치와 이상치를 처리하는 방법을 정리하고, 간단한 실습을 통해 데이터 전처리 과정을 살펴보았다.

---

# 📌 데이터 전처리에서 결측치와 이상치가 중요한 이유

머신러닝 프로젝트는 일반적으로 다음과 같은 순서로 진행된다.

```text
데이터 수집

↓

EDA(탐색적 데이터 분석)

↓

결측치 처리

↓

이상치 처리

↓

Feature Engineering

↓

Train/Test Split

↓

모델 학습

↓

성능 평가
```

결측치와 이상치를 적절히 처리하지 않으면 모델이 데이터의 패턴을 제대로 학습하지 못하거나 성능이 크게 저하될 수 있다.

---

# 1️⃣ 결측치(Missing Value)

## 결측치란?

결측치(Missing Value)는 **값이 존재하지 않는 데이터**를 의미한다.

예를 들어 고객 정보를 수집했다고 가정해 보자.

| 이름 | 나이 | 연봉 |
|------|----:|----:|
| 김철수 | 28 | 4500 |
| 이영희 | NaN | 5200 |
| 박민수 | 35 | NaN |

여기서 `NaN` 값이 결측치이다.

결측치는 데이터 입력 오류, 설문 미응답, 시스템 오류 등 다양한 원인으로 발생할 수 있다.

---

## 결측치 확인

Pandas에서는 `isnull()` 또는 `isna()`를 이용하여 결측치를 쉽게 확인할 수 있다.

```python
df.isnull().sum()
```

### 출력 예시

```text
Age         177
Cabin       687
Embarked      2
```

각 컬럼에 존재하는 결측치 개수를 확인할 수 있다.

### 📷 실행 결과

아래는 `isnull().sum()`을 이용하여 Titanic 데이터셋의 결측치를 확인한 결과이다.

![결측치 확인](/images/ml/결측치확인.png)

---

## 결측치 처리 방법

결측치를 처리하는 방법은 크게 세 가지로 나눌 수 있다.

### ① 삭제(Delete)

결측치가 포함된 행 또는 열을 삭제한다.

```python
df.dropna()
```

특정 컬럼을 삭제할 수도 있다.

```python
df.drop(columns=["Cabin"])
```

> 결측치 비율이 매우 높거나 데이터가 충분한 경우 많이 사용된다.

---

### ② 대체(Imputation)

결측치를 평균, 중앙값, 최빈값 등으로 채우는 방법이다.

평균으로 대체

```python
df["Age"].fillna(df["Age"].mean())
```

중앙값으로 대체

```python
df["Age"].fillna(df["Age"].median())
```

### 📷 실행 결과

결측치를 중앙값으로 대체한 이후 결측치를 다시 확인해보았다.

![결측치 처리 후 확인](/images/ml/결측치처리후확인.png)

---

### ③ 머신러닝 기반 대체

단순 평균으로 처리하기 어려운 경우에는 머신러닝 모델을 이용하기도 한다.

대표적으로

- KNN Imputer
- Iterative Imputer(MICE)

등이 사용된다.

다만 입문 단계에서는 평균, 중앙값, 최빈값을 이용한 대체가 가장 많이 사용된다.

---

# 📌 결측치를 무조건 삭제하면 안 되는 이유

결측치가 있다고 해서 항상 삭제하는 것이 좋은 것은 아니다.

예를 들어 데이터가 500개뿐인데 100개를 삭제하면 학습에 사용할 데이터가 크게 줄어든다.

따라서

- 데이터 개수
- 결측치 비율
- 결측 원인

등을 함께 고려하여 처리 방법을 선택해야 한다.

# 2️⃣ 이상치(Outlier)

## 이상치란?

이상치(Outlier)는 다른 데이터와 비교했을 때 **지나치게 크거나 작은 값**을 의미한다.

예를 들어

| 연봉 |
|----:|
|3200|
|3500|
|4100|
|3900|
|850000|

850000은 다른 값들과 비교했을 때 매우 큰 값이므로 이상치라고 볼 수 있다.

이상치는

- 입력 오류
- 측정 오류
- 특수한 상황

등 다양한 이유로 발생할 수 있다.

---

## 이상치가 문제가 되는 이유

이상치가 존재하면

- 평균이 크게 왜곡될 수 있고
- 모델이 특정 데이터에 과도하게 영향을 받을 수 있으며
- 예측 성능이 저하될 수 있다.

따라서 모델 학습 전에 이상치를 확인하는 과정이 중요하다.

---

# 📌 Box Plot을 이용한 이상치 확인

가장 많이 사용하는 방법은 **Box Plot(상자 그림)** 이다.

```python
sns.boxplot(x=df["fare"])
```

### 📷 실행 결과

![Fare 이상치 확인](/images/ml/Fare이상치확인.png)

위는 Titanic 데이터셋의 Fare 컬럼을 Box Plot으로 시각화한 결과이다.

상자의 수염(Whisker)을 벗어난 점(○)들은 IQR 기준으로 판단한 이상치 후보를 의미한다.

특히 Fare 값이 약 100 이상인 일부 데이터와 약 512 부근의 데이터는 다른 데이터에 비해 매우 큰 값을 가지므로 극단적인 이상치로 확인할 수 있었다.

---

# 📌 IQR(Interquartile Range)

Box Plot은 IQR을 이용하여 이상치를 판단한다.

IQR은

$$
IQR = Q3 - Q1
$$

으로 계산한다.

이후

$$
Lower = Q1 - 1.5 \times IQR
$$

$$
Upper = Q3 + 1.5 \times IQR
$$

범위를 벗어나는 데이터를 이상치 후보로 판단한다.

### 📷 IQR 계산

![IQR 계산](/images/ml/IQR및이상치범위계산.png)

# 📌 이상치 처리 방법

## ① 제거(Delete)

입력 오류나 비정상적인 데이터라면 삭제하는 것이 일반적이다.

예를 들어

```python
df_clean = df[
    (df["fare"] >= lower) &
    (df["fare"] <= upper)
]
```

처럼 IQR 범위를 벗어난 데이터를 제거할 수 있다.

---

### 📷 이상치 제거 결과

IQR 기준을 적용하여 이상치를 제거한 결과, 데이터 개수는 **891개에서 775개**로 감소하였다.

![이상치 제거 후](/images/ml/제거후boxplot.png)

극단적으로 큰 Fare 값은 제거되었으며 전체 데이터 분포도 이전보다 안정적인 형태를 보인다.

다만 Box Plot에는 여전히 일부 점(○)이 표시되는 것을 확인할 수 있다.

이는 이상치 제거가 실패한 것이 아니라, **이상치를 제거한 이후 데이터 분포가 달라지면서 새로운 IQR 기준이 적용되었기 때문**이다.

즉, Box Plot은 현재 데이터의 분포를 기준으로 이상치를 다시 판단하기 때문에 일부 점이 계속 표시될 수 있다.

> **핵심 정리**
>
> 극단적인 이상치는 제거되었지만, Box Plot은 현재 데이터의 분포를 기준으로 이상치를 다시 판단하기 때문에 일부 점이 계속 표시될 수 있다.

---

## ② 대체(Replacement)

삭제하기 어려운 경우에는 평균이나 중앙값 등으로 대체하기도 한다.

---

## ③ 변수화(Transformation)

이상치 자체가 중요한 의미를 갖는 경우에는 삭제하지 않고 새로운 변수로 활용하기도 한다.

예를 들어

```text
연봉

↓

연봉 구간
```

처럼 범주형 변수로 변환하여 사용할 수 있다.


# 📊 결측치와 이상치 비교

| 구분 | 결측치 | 이상치 |
|------|---------|---------|
| 의미 | 값이 없음 | 비정상적으로 큰 값 또는 작은 값 |
| 대표 예 | NaN | 음수 요금, 비정상적인 연봉 |
| 확인 방법 | isnull(), isna() | Box Plot, 통계 분석 |
| 대표 처리 | 삭제, 대체 | 제거, 대체, 변수화 |

---

# 💡 공부하면서 느낀 점

처음에는 결측치와 이상치를 단순히 삭제하는 작업이라고 생각했다.

하지만 실제 데이터를 다루면서 중요한 것은 데이터를 얼마나 많이 삭제하느냐가 아니라, **왜 해당 데이터를 처리하는지에 대한 근거를 갖는 것**이라는 점을 알게 되었다.

또한 이상치는 무조건 제거하는 것이 아니라 데이터의 의미를 먼저 파악한 뒤 제거, 대체, 변수화 중 적절한 방법을 선택해야 한다는 점도 함께 이해할 수 있었다.

결국 데이터 전처리는 단순한 정리 작업이 아니라 **데이터를 이해하고 머신러닝 모델이 올바르게 학습할 수 있도록 준비하는 과정**이라는 점을 다시 한 번 느낄 수 있었다.
