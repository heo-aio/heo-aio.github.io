# 🔄 [OS] 핵심 운영체제 - 프로세스(Process) 개념과 구조 정리

프로세스(Process)는 운영체제를 공부하면서 가장 먼저 마주하게 되는 핵심 개념 중 하나이다. 처음에는 프로그램, 프로세스, 스레드의 차이가 헷갈렸지만, 개념을 정리하면서 각각의 역할을 조금 더 명확하게 이해할 수 있었다.

이번 글에서는 프로세스의 정의와 메모리 구조를 정리하고, 공부하며 이해한 내용을 함께 기록해 보려고 한다.

---

## 📌 목차

1. 프로세스의 개념 (프로그램 vs 프로세스 vs 스레드)
2. 프로세스의 메모리 구조
3. 공부하며 이해한 내용
4. 용어 정리 (컴파일이란?)

---

# 01. 프로세스의 개념

운영체제를 공부할 때 가장 먼저 구분해야 하는 개념이 프로그램, 프로세스, 스레드이다.

## 1) 프로그램 (Program)

* 특정 작업을 수행하기 위한 명령어들의 집합이다.
* 디스크나 보조기억장치에 저장된 정적인 상태를 의미한다.
* 아직 실행되지 않았기 때문에 CPU나 메모리를 사용하지 않는다.

예를 들어 컴퓨터에 설치된 크롬(Chrome)은 프로그램이다.

---

## 2) 프로세스 (Process)

* 프로그램이 실제로 실행되고 있는 상태이다.
* 운영체제로부터 CPU, 메모리 등의 자원을 할당받아 동작한다.
* 실행 중인 프로그램 하나하나를 프로세스라고 부른다.

예를 들어 크롬을 실행하면 프로세스가 생성된다.

---

## 3) 스레드 (Thread)

* 프로세스 내부에서 실제 작업을 수행하는 실행 단위이다.
* 하나의 프로세스는 하나 이상의 스레드를 가진다.
* 여러 스레드가 동시에 동작하며 작업을 처리할 수 있다.

---

### 💡 프로세스와 스레드의 차이

**프로세스**

* 독립적인 메모리 공간을 가진다.
* 다른 프로세스와 메모리를 직접 공유하지 않는다.
* 안정성은 높지만 자원 사용량이 크다.

**스레드**

* 같은 프로세스 내부에서 생성된다.
* 프로세스의 메모리 영역을 공유한다.
* 자원을 효율적으로 사용할 수 있지만 동기화 문제가 발생할 수 있다.

---

# 02. 프로세스의 메모리 구조

프로세스가 생성되면 운영체제는 메모리를 여러 영역으로 나누어 관리한다.

대표적으로 Stack, Heap, Data, Code 영역이 존재한다.

| 영역    | 설명                                         |
| ----- | ------------------------------------------ |
| Stack | 함수 호출 시 생성되는 지역 변수, 매개변수, 복귀 주소 등이 저장되는 영역 |
| Heap  | 실행 중 동적으로 생성되는 데이터가 저장되는 영역                |
| Data  | 전역 변수, 정적 변수 등이 저장되는 영역                    |
| Code  | CPU가 실행할 기계어 명령어가 저장되는 영역                  |

---

## 🔍 Code 영역

Code 영역은 Text 영역이라고도 불린다.

컴파일된 기계어 코드가 저장되며 프로그램 실행 중 임의로 수정되어서는 안 된다.

따라서 운영체제는 해당 영역을 읽기 전용(Read-Only)으로 관리한다.

이를 통해 프로그램의 안정성과 보안성을 확보할 수 있다.

---

# 03. 💭 공부하며 이해한 내용

처음에는 프로그램과 프로세스를 같은 의미로 생각했다.

하지만 공부를 진행하면서 프로그램은 단순히 저장된 파일이고, 프로세스는 실제로 실행 중인 상태라는 점을 이해하게 되었다.

특히 "프로그램은 설계도이고 프로세스는 실제 작업자"라고 생각하니 개념이 훨씬 쉽게 와닿았다.

또한 같은 프로그램이라도 여러 번 실행하면 여러 개의 프로세스가 생성될 수 있다는 점이 흥미로웠다.

예를 들어 크롬을 여러 개 실행하면 프로그램은 하나지만 프로세스는 여러 개가 존재할 수 있다.

---

메모리 구조 역시 처음에는 단순 암기 대상처럼 느껴졌다.

Stack, Heap, Data, Code 영역의 이름을 외우는 데 집중했지만, 왜 영역을 나누어 관리하는지 이해하지 못했다.

공부를 진행하면서 운영체제가 프로그램을 효율적이고 안전하게 관리하기 위해 메모리를 분리한다는 점을 알게 되었다.

예를 들어 함수 내부에서만 사용하는 데이터는 Stack에 저장하고, 실행 중 크기가 결정되는 데이터는 Heap에 저장한다.

이렇게 역할을 나누어 관리하면 메모리를 보다 효율적으로 사용할 수 있다.

결국 메모리 구조는 단순 암기용 개념이 아니라 운영체제가 자원을 관리하기 위한 설계라는 점이 인상 깊었다.

---

# 04. 🛠️ 용어 정리

## 컴파일 (Compile)

개발자가 작성한 소스 코드를 컴퓨터가 이해할 수 있는 기계어로 변환하는 과정이다.

---

## 컴파일러 (Compiler)

컴파일 작업을 수행하는 프로그램이다.

대표적으로 GCC, Clang, javac 등이 있다.

---

# 📚 마무리

이번 정리를 통해 프로그램, 프로세스, 스레드의 차이를 다시 한번 정리할 수 있었다.

특히 프로세스의 메모리 구조는 단순 암기보다 각 영역이 왜 존재하는지 이해하는 것이 중요하다고 느꼈다.

운영체제는 결국 제한된 자원을 효율적으로 관리하기 위한 소프트웨어이며, 프로세스 역시 그 핵심 개념 중 하나라는 점을 기억해 두려고 한다.
