# 🔄 운영체제 핵심 개념 정리 - 프로세스(Process)와 메모리 구조

운영체제를 학습하다 보면 가장 먼저 접하게 되는 개념 중 하나가 **프로세스(Process)** 이다.

프로세스는 단순히 "실행 중인 프로그램"이라는 의미를 넘어 CPU 스케줄링, 메모리 관리, 프로세스 간 통신(IPC) 등 운영체제 전반의 핵심 기능과 연결되는 중요한 개념이다.

이번 글에서는 프로세스의 개념과 메모리 구조를 중심으로 정리해본다.

# 01. 📖 프로세스(Process)란?

프로세스(Process)는 **실행 중인 프로그램**을 의미한다.

프로그램이 단순히 저장장치에 존재하는 파일이라면, 프로세스는 운영체제로부터 CPU와 메모리를 할당받아 실제로 동작하는 실행 단위이다.

예를 들어 Chrome 실행 파일은 프로그램이지만, 사용자가 Chrome을 실행하는 순간 운영체제는 새로운 프로세스를 생성하여 실행 환경을 제공한다.

---

# 02. 🔍 프로그램 vs 프로세스 vs 스레드

운영체제를 공부할 때 가장 헷갈리는 개념 중 하나가 프로그램, 프로세스, 스레드의 차이이다.

| 구분      | 설명               |
| ------- | ---------------- |
| Program | 저장장치에 존재하는 실행 파일 |
| Process | 실행 중인 프로그램       |
| Thread  | 프로세스 내부의 실행 단위   |

---

### Program

* 디스크에 저장된 정적인 상태
* CPU와 메모리를 사용하지 않음

---

### Process

* 실행 중인 프로그램
* 운영체제가 자원을 할당하여 관리

---

### Thread

* 프로세스 내부에서 실제 작업 수행
* 하나의 프로세스는 여러 스레드를 가질 수 있음

---

## 💡 프로세스와 스레드의 차이

### 프로세스

✅ 독립적인 메모리 공간 사용

✅ 다른 프로세스와 메모리 분리

✅ 안정성 높음

❌ 생성 비용이 큼

---

### 스레드

✅ 메모리 공유 가능

✅ 생성 비용이 적음

✅ 빠른 작업 처리 가능

❌ 동기화 문제 발생 가능

---

# 03. ⚙️ 프로세스의 메모리 구조

프로세스가 생성되면 운영체제는 메모리를 목적에 따라 여러 영역으로 나누어 관리한다.

대표적으로 다음 네 가지 영역으로 구성된다.

```text
높은 주소
┌─────────────┐
│   Stack     │
├─────────────┤
│             │
│    Heap     │
│             │
├─────────────┤
│    Data     │
├─────────────┤
│    Code     │
└─────────────┘
낮은 주소
```

---

| 영역    | 역할              |
| ----- | --------------- |
| Code  | 실행 명령어 저장       |
| Data  | 전역 변수, 정적 변수 저장 |
| Heap  | 동적 메모리 할당       |
| Stack | 함수 호출 정보 저장     |

---

## 🧩 Code 영역

Code(Text) 영역은 프로그램의 실행 명령어가 저장되는 공간이다.

컴파일된 기계어 코드가 저장되며 프로그램 실행 중 변경되지 않도록 일반적으로 읽기 전용(Read Only)으로 관리된다.

---

## 🧩 Data 영역

전역 변수(Global Variable)와 정적 변수(Static Variable)가 저장되는 공간이다.

프로그램 시작 시 메모리에 할당되며 종료될 때까지 유지된다.

---

## 🧩 Heap 영역

실행 중 동적으로 생성되는 데이터를 저장하는 공간이다.

필요한 만큼 메모리를 할당받을 수 있지만 사용 후 적절히 해제하지 않으면 메모리 누수(Memory Leak)가 발생할 수 있다.

---

## 🧩 Stack 영역

함수 호출 시 생성되는 지역 변수, 매개변수, 복귀 주소 등을 저장한다.

후입선출(LIFO) 구조로 동작하며 함수 실행이 종료되면 자동으로 메모리가 해제된다.

---

# 04. 🚀 실제 예시로 이해하기

다음과 같은 상황을 생각해보자.

```python
def hello():
    name = "ChatGPT"

hello()
```

함수가 호출되면

1. `hello()` 함수 정보가 Stack에 저장된다.
2. 지역 변수 `name` 역시 Stack에 생성된다.
3. 함수가 종료되면 Stack 영역의 데이터가 제거된다.

반면 다음과 같은 객체는 Heap 영역에 생성된다.

```python
user = {
    "name": "ChatGPT",
    "age": 10
}
```

실행 중 생성되는 객체는 일반적으로 Heap 영역에 저장된다.

# 05. 💼 면접 포인트

면접에서 자주 나오는 질문들이다.

### Q1. 프로그램과 프로세스의 차이는?

> 프로그램은 저장된 파일이고, 프로세스는 실행 중인 프로그램이다.

---

### Q2. 프로세스와 스레드의 차이는?

> 프로세스는 독립된 메모리를 사용하고, 스레드는 같은 프로세스 내에서 메모리를 공유한다.

---

### Q3. Heap과 Stack의 차이는?

| Stack    | Heap        |
| -------- | ----------- |
| 자동 관리    | 수동 또는 GC 관리 |
| 빠름       | 상대적으로 느림    |
| 지역 변수 저장 | 동적 데이터 저장   |

---

### Q4. Stack Overflow란?

> 함수가 과도하게 호출되거나 재귀 호출이 반복되어 Stack 공간이 부족해지는 현상이다.

---

# 06. 📝 핵심 정리

✅ 프로그램은 저장된 실행 파일이다.

✅ 프로세스는 실행 중인 프로그램이다.

✅ 스레드는 프로세스 내부의 실행 단위이다.

✅ 프로세스는 Code, Data, Heap, Stack 영역으로 구성된다.

✅ Heap은 동적 메모리, Stack은 함수 호출 정보를 관리한다.

✅ 프로세스 구조는 CPU 스케줄링, 가상 메모리, IPC 등의 운영체제 핵심 개념의 기초가 된다.
