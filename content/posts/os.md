# 🔄 운영체제 핵심 개념 정리 - 프로세스(Process)와 메모리 구조

운영체제를 공부하면 가장 먼저 등장하는 개념이 프로세스(Process)이다.

사실 처음에는 프로세스가 그냥 "실행 중인 프로그램" 정도라고만 알고 있었는데, 공부를 계속하다 보니 CPU 스케줄링, 메모리 관리, IPC 같은 운영체제 핵심 개념들이 전부 프로세스와 연결되어 있었다.

그래서 이번에는 프로세스와 메모리 구조를 중심으로 정리해보려고 한다.

---

# 📖 프로세스(Process)란?

프로세스는 간단히 말하면 **실행 중인 프로그램**이다.

프로그램은 디스크에 저장된 파일일 뿐이지만,

실제로 실행되는 순간 운영체제로부터 CPU와 메모리를 할당받고 프로세스가 된다.

예를 들어 Chrome 설치 파일은 프로그램이고,

실행된 Chrome은 프로세스이다.

---

## 🤔 처음에 헷갈렸던 부분

처음에는 프로그램과 프로세스를 거의 같은 개념으로 생각했다.

하지만 다시 정리해보니 차이가 명확했다.

| 구분      | 의미         |
| ------- | ---------- |
| Program | 저장된 실행 파일  |
| Process | 실행 중인 프로그램 |

결국 프로세스는

> 프로그램 + 운영체제가 할당한 자원

이라고 이해하면 훨씬 편했다.

---

# 🔍 프로그램 vs 프로세스 vs 스레드

운영체제를 공부할 때 가장 자주 등장하는 비교이다.

| 구분      | 설명            |
| ------- | ------------- |
| Program | 저장장치에 존재하는 파일 |
| Process | 실행 중인 프로그램    |
| Thread  | 프로세스 내부 실행 단위 |

---

## 프로세스

✅ 독립된 메모리 공간 사용

✅ 안정성 높음

❌ 생성 비용 큼

---

## 스레드

✅ 메모리 공유

✅ 생성 비용 적음

❌ 동기화 문제 발생 가능

---

## 🤔 내가 이해한 방식

프로세스는 각각 독립된 집이라고 생각하면 편했다.

반면 스레드는 같은 집 안에 사는 가족 같은 느낌이다.

그래서 스레드는 자원을 공유할 수 있지만,

그만큼 충돌도 발생할 수 있다.

---

# ⚙️ 프로세스의 메모리 구조

프로세스가 생성되면 운영체제는 메모리를 여러 영역으로 나누어 관리한다.

```text
높은 주소

┌─────────────┐
│   Stack     │
├─────────────┤
│             │
│    Heap     │
│             │
├─────────────┤
│    Data     │
├─────────────┤
│    Code     │
└─────────────┘

낮은 주소
```

---

## 한눈에 보기

| 영역    | 역할           |
| ----- | ------------ |
| Code  | 실행 코드 저장     |
| Data  | 전역 변수, 정적 변수 |
| Heap  | 동적 할당 메모리    |
| Stack | 함수 호출 정보     |

---

# 🧩 Code 영역

프로그램의 실행 명령어가 저장되는 공간이다.

보통 읽기 전용(Read Only)으로 관리된다.

---

# 🧩 Data 영역

전역 변수(Global Variable),

정적 변수(Static Variable)가 저장된다.

프로그램 시작부터 종료까지 유지된다.

---

# 🧩 Heap 영역

실행 중 동적으로 생성되는 데이터를 저장한다.

```python
user = {
    "name": "ChatGPT",
    "age": 10
}
```

이런 객체들은 일반적으로 Heap 영역에 생성된다.

---

# 🧩 Stack 영역

함수 호출 시 생성되는 지역 변수와 매개변수가 저장된다.

```python
def hello():
    name = "ChatGPT"

hello()
```

함수가 종료되면 Stack에 저장된 정보도 함께 제거된다.

---

## 📌 Stack과 Heap은 꼭 구분하기

운영체제 공부하면서 가장 많이 들은 질문이

> Heap과 Stack의 차이는?

이었다.

정리하면 아래 정도는 기억하고 가면 좋을 것 같다.

| Stack | Heap        |
| ----- | ----------- |
| 자동 관리 | 직접 관리 또는 GC |
| 빠름    | 상대적으로 느림    |
| 지역 변수 | 동적 데이터      |

---

# 🤔 공부하면서 느낀 점

처음에는 Code, Data, Heap, Stack을 그냥 암기하려고 했다.

그런데 계속 까먹었다.

오히려

* 함수 호출 → Stack
* 객체 생성 → Heap
* 전역 변수 → Data

처럼 실제 코드와 연결해서 보니까 훨씬 기억에 오래 남았다.

운영체제는 암기 과목처럼 보이지만,

생각보다 "왜 이렇게 설계했을까?"를 이해하는 게 더 중요한 것 같다.

---

# 🚀 주기적으로 다시 보기

* 프로그램 = 저장된 파일
* 프로세스 = 실행 중인 프로그램
* 스레드 = 프로세스 내부 실행 단위
* Code = 실행 코드
* Data = 전역 변수, 정적 변수
* Heap = 동적 메모리
* Stack = 함수 호출 정보
* 프로세스는 CPU 스케줄링과 메모리 관리의 기본 단위

운영체제를 공부할수록 느끼는 건,

프로세스와 메모리 구조를 제대로 이해해야 이후에 나오는 스레드, IPC, 스케줄링도 자연스럽게 연결된다는 점이다.

아직은 익숙하지 않지만 계속 반복해서 보다 보면 운영체제 전체 흐름이 조금씩 보일 것 같다.
